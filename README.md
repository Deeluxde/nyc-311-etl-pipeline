# NYC 311 ETL Pipeline

## End-to-End ETL Pipeline Using Python, Pandas, SQLite, Docker, PyTest, GitHub Actions, and Git

---

## Project Overview

This project demonstrates an end-to-end Extract, Transform, Load (ETL) pipeline built in Python to process NYC 311 Service Request data.

The objective of this project was to build a modular ETL pipeline that extracts raw data, cleans and validates it, and loads the transformed data into a SQLite database. Throughout the development process, the project evolved from a single Python script into a structured pipeline with separate extraction, transformation, validation, and loading modules.

To improve maintainability and reliability, additional features such as centralized logging, environment-based configuration, automated unit testing, Docker containerization, and GitHub Actions Continuous Integration (CI) were implemented.

## ETL Pipeline

### Extract

- Read raw CSV data using Pandas
- Validate source file
- Log extraction process

### Transform

- Removed duplicate records
- Standardized column names
- Trimmed whitespace
- Converted date fields to datetime format
- Selected production-ready columns
- Renamed columns to match the database schema

### Validate

- Verified required columns
- Checked for missing values
- Confirmed dataset integrity before loading

### Load

- Created SQLite database
- Executed database schema
- Loaded transformed data
- Verified successful insert
- Logged completion of the ETL process

## Running the Project

### Install Dependencies

```bash
pip install -r requirements.txt
```

### Run the ETL Pipeline

```bash
python3 main.py
```

### Run Unit Tests

```bash
pytest
```

### Build the Docker Image

```bash
docker build -t nyc-311-etl-pipeline .
```

### Run the Docker Container

```bash
docker run --rm \
--env-file .env \
-v "$PWD/data:/app/data" \
-v "$PWD/output:/app/output" \
nyc-311-etl-pipeline
```

## Development Notes

This project started as a simple ETL assignment and was gradually enhanced into a modular data engineering project.

During development, the pipeline was refactored into separate extraction, transformation, validation, and loading modules. Centralized logging, environment-based configuration, automated testing with PyTest, Docker containerization, and GitHub Actions Continuous Integration were added to improve maintainability, reproducibility, and code quality.

Developing this project also provided practical experience troubleshooting Python module imports, configuring Docker, managing environment variables, and implementing automated testing workflows.
