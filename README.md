# NYC 311 ETL Pipeline

## Project Overview

This project demonstrates an end-to-end ETL (Extract, Transform, Load) pipeline using Python and SQLite to process NYC 311 Service Request data.

The project extracts a publicly available dataset from NYC Open Data, performs data cleaning and transformation using Python and Pandas, loads the processed data into a SQLite database, and executes SQL queries to generate operational insights.

---

## Dataset

**Dataset Name**

NYC 311 Service Requests from 2020 to Present

**Dataset Used**

Filtered records from **January 1, 2026 through March 31, 2026**.

**Source**

NYC Open Data

**Dataset URL**

https://data.cityofnewyork.us/Social-Services/311-Service-Requests-from-2020-to-Present/erm2-nwe9

---

## Why I Selected This Dataset

I selected this dataset because it contains real-world operational data generated by New York City's 311 service system. The filtered dataset includes over one million service requests, providing sufficient volume to demonstrate data engineering concepts such as data extraction, transformation, relational database design, SQL querying, and exploratory analysis while remaining manageable for development and testing.

## Planned ETL Workflow

1. Extract the raw CSV dataset.
2. Clean and transform the data using Pandas.
3. Load the transformed data into SQLite.
4. Execute SQL queries to answer business questions.
5. Generate summary statistics and visualizations.
